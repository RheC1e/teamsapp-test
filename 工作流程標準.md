## 🚀 標準測試 Demo 流程

### 流程概述

這是我們最喜歡的測試 demo 流程，簡單、清晰、高效：

1. **您創建 Git 倉庫** → 2. **Vercel 連結** → 3. **告訴我 URL**

---

## 詳細步驟

### 步驟 1：創建 GitHub 倉庫

**您要做的事：**
1. 點擊連結：https://github.com/new
2. 填寫資訊：
   - Repository name: `專案名稱`（例如：`teamsapp-test`）
   - Description: 專案描述
   - 選擇 Private 或 Public
   - **不要**勾選任何選項（README、.gitignore、License）
3. 點擊「Create repository」

**完成後：**
- 告訴我「GitHub 已創建」或「GitHub 倉庫已創建」
- 我會立即推送代碼到 GitHub

---

### 步驟 2：部署到 Vercel

**您要做的事：**
1. 點擊連結：https://vercel.com/new
2. 點擊「Continue with GitHub」登入（使用 rhemaluis125@gmail.com）
3. 在倉庫列表中，找到並選擇您的專案（例如：`RheC1e/teamsapp-test`）
4. 點擊「Import」
5. 確認設定（通常會自動偵測正確）：
   - Framework Preset: Vite（或自動偵測）
   - Build Command: `npm run build`
   - Output Directory: `dist`
6. 點擊「Deploy」
7. 等待約 1-2 分鐘部署完成
8. **記下部署後的 URL**（例如：`teamsapp-test-xxx.vercel.app`）

**完成後：**
- 告訴我「Vercel 部署 URL 是：xxx.vercel.app」
- 我會立即更新相關設定

---

### 步驟 3：告訴我 URL

**您要做的事：**
- 直接告訴我 Vercel 部署 URL

**我會立即做的事：**
- ✅ 更新 manifest.json 中的 URL
- ✅ 更新相關配置檔案
- ✅ 重新生成 Teams 應用程式包（如果需要）
- ✅ 檢查所有設定
- ✅ 確認部署狀態

---

## ✅ 流程優點

### 為什麼這個流程很好？

1. **簡單明確**
   - 只需要點擊兩個網址
   - 不需要在終端機執行命令
   - 不需要執行腳本

2. **分工清楚**
   - 您負責：創建倉庫、部署、提供 URL
   - 我負責：推送代碼、更新設定、檢查配置

3. **高效快速**
   - 每個步驟都有明確的目標
   - 不會混淆或失焦
   - 專注於解決問題

4. **易於重複**
   - 每次都用同樣的流程
   - 不需要重新學習
   - 建立合作默契

## 🎯 專案原則

### 我們專注於：

- ✅ **解決問題**：專注於當前的任務目標
- ✅ **簡單直接**：不添加不必要的複雜性
- ✅ **清晰明確**：每個步驟都有明確的目的
- ✅ **建立默契**：每次合作都更順暢

### 我們避免：

- ❌ 添加與當前任務無關的內容（如遊戲化、願景等）
- ❌ 多工失焦
- ❌ 讓您執行複雜的技術操作
- ❌ 混淆或跳步
